Evidence Lab Local Scenarios
============================

These are local/manual scenarios for the public evidence lab UI. They replace the evidence-lab-specific test file so the prototype can be exercised manually in Vercel.

Route
-----
/evidence-lab

Manual checks
-------------
1. Open /evidence-lab without query params.
   Expected: data-source catalog and upload panel are visible. No assessment dashboard is open yet.

2. Open /evidence-lab?fixture=corroborated-acme-client-concern.
   Expected: assessment dashboard opens for the fixture and shows corroborated claim kinds.

3. Open /evidence-lab?fixture=conflict-acme-timeline-risk&source=source-acme-current-risk-cleared-note.
   Expected: source reader opens with the current CRM note and shows a contradicting implementation-risk claim.

4. Upload one or more .txt/.md/.json files.
   Expected: uploaded sources are parsed locally in the browser, a custom bundle is assessed, and claims are inferred with deterministic keyword heuristics.

5. Upload text containing words like implementation, timeline, pricing, stakeholder, renewal, decision, confidential, partner.
   Expected: the uploaded source receives matching claim kinds and sensitivity values based on those keywords.

6. After uploading a realistic email, wait for automatic GPT-5.5 extraction.
   Expected: the source mode changes from Keyword parsed to AI extracted, source claims are replaced by OpenAI's structured extraction, and the deterministic scoring reruns on the normalized output.

7. Upload northstar-renewal-thread.eml from static/evidence-lab-samples and wait for AI extraction.
   Expected: pricing context is treated as stale/validation-needed, implementation timeline is treated as current context, cautions mention not using pricing as the main blocker without Finance confirmation, and missing context stays focused to a few blocking validation questions.

8. After extraction, inspect the grouped claim cards.
   Expected: current implementation/timeline/stakeholder claims appear under usable context, stale pricing/procurement claims appear under needs validation, and contradiction-style claims appear under do not state directly.

9. Open a source and click See full text.
   Expected: uploaded source text is hidden by default and appears only after clicking See full text.

10. Open View detailed analysis.
    Expected: raw source-level scores, score explanations, weaknesses, and local truth metadata appear in the collapsed details section.

Important notes
---------------
- Keyword-parsed uploads are browser-local only.
- Uploaded file contents are not persisted to Convex.
- Uploaded file contents are sent to OpenAI by default after upload in this public prototype.
- AI extraction uses GPT-5.5 with low reasoning effort on the server.
- The final ranking/scoring is still deterministic after extraction.
- Claim grouping is a UI interpretation layer; it does not replace the deterministic ranking engine.
